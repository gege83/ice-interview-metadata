Track metadata application demo

This page is just for demonstration. There are 3 users that has been created for testing purposes: artist1, artist2 and user artist1 and artist2 are with role ARTIST, they can create aliases and manage their tracks. While user has USER role, allows it to list artists and tracks but not allowed to create or modify one.

The database is an H2 inmemory database, so every restart will reset the data. To quickly generate some data please click on `Generate base data` button.

To create a new artist alias you can use the POST /artists endpoint. To update an existing one you need to use the PUT /artists/{id}. To list all aliases for a user use the GET /artists endpoint.

To create a new track metadata you can use the POST /track endpoint. To update an existing one you need to use the PUT /track/{id}. To list all aliases for a user use the GET /tracks?artistId={artistId} endpoint.

Artist of the day endpoint is GET /public/artist-of-the-day. The endpoint will return the same artist alias for a day for all users.
